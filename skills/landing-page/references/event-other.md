# Event, Waitlist, and Other Landing Page Types

## Event / Conference Landing Page

### Sections
1. **Hero**: Event name + date + location + primary CTA ("Register now" / "Get tickets")
2. **Key details**: Date, time, location (with map embed or link), format (in-person/virtual/hybrid)
3. **Speakers / Lineup**: Photo + name + title + topic. Grid or carousel.
4. **Schedule / Agenda**: Timeline format. Collapsible by day if multi-day.
5. **Tickets / Pricing**: Tier cards (Early bird, General, VIP). Include what's in each tier.
6. **Sponsors**: Logo grid, tiered by sponsorship level
7. **FAQ**: Refund policy, dress code, accessibility, parking, livestream availability
8. **Final CTA**: Countdown timer + register button

### Patterns
- Countdown timer in hero creates urgency
- "Early bird" pricing with deadline shown
- Speaker cards link to individual bios or talk descriptions
- Past event photos/videos build credibility for recurring events

## Waitlist / Coming Soon Landing Page

### Sections
1. **Hero**: Product name + one-line teaser + email capture form
2. **Teaser**: 2-3 bullet points on what's coming (vague enough to intrigue, specific enough to be credible)
3. **Social proof**: "Join 2,000+ on the waitlist" (counter, real or approximate)
4. **Optional**: Founder note, behind-the-scenes preview, early access perks

### Patterns
- Single-field email capture (minimize friction)
- Progress indicator: "You're #847 in line"
- Referral program: "Move up the list by sharing"
- Minimal navigation — one page, one action

## Nonprofit / Cause Landing Page

### Sections
1. **Hero**: Emotional headline + impactful image + donate CTA
2. **The problem**: Statistics + human story
3. **The solution**: What the org does, in 3 steps
4. **Impact**: Metrics, before/after, beneficiary stories
5. **How to help**: Donate / Volunteer / Share — multiple CTAs
6. **Transparency**: Where donations go (pie chart or breakdown)
7. **Final CTA**: Repeat donate with suggested amounts ($25, $50, $100, Custom)

---

## App Download Landing Page

### Sections
1. **Hero**: App name + tagline + App Store/Play Store badges + phone mockup
2. **Features**: 3-4 with phone screen showing each feature
3. **Reviews**: App store ratings + select user reviews
4. **Download CTA**: Repeated badges, QR code for desktop visitors
5. **Footer**: Minimal — privacy policy, support link

### Patterns
- Phone mockup is essential — show the actual app
- QR code for visitors on desktop to quickly get to app store
- Star rating prominently displayed
- "Free" or "No ads" if applicable — call it out in hero
