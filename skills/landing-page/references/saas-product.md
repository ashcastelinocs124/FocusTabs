# SaaS / Product Landing Page Reference

## Required Sections (in order)

### 1. Hero
- **Headline**: One clear value proposition. Max 10 words. Answer "What do I get?"
- **Subheadline**: 1-2 sentences expanding on the how/why
- **Primary CTA**: High-contrast button, action verb ("Start free trial", "Get started free")
- **Secondary CTA**: Low-commitment alternative ("See demo", "Watch video")
- **Social proof badge**: Inline — "Trusted by 10,000+ teams" or logo strip
- **Hero visual**: Product screenshot, demo video, or interactive preview — NOT a stock photo

```
Pattern: Problem-agitate-solve in 3 lines
  Headline  → names the outcome
  Sub       → acknowledges the pain
  CTA       → offers the fix
```

### 2. Logo Bar / Social Proof Strip
- 4-6 recognizable logos, grayscale, evenly spaced
- Optional: "Trusted by teams at..." prefix
- Place immediately below hero to build credibility before they scroll

### 3. Problem Statement
- 2-3 short pain points the target user faces
- Use the user's language, not jargon
- Format: icon + short title + one-line description
- Goal: make the reader nod "yes, that's me"

### 4. Features / Benefits (3-4 max)
- Lead with benefit, not feature name
- Pattern: `Benefit headline → 1-sentence explanation → supporting visual`
- Use alternating left/right layout or card grid
- Each feature should map to a pain point from section 3

### 5. How It Works
- 3 steps maximum (numbered)
- Pattern: `Step number → Action verb title → One sentence`
- Include a visual per step or a single flow diagram
- End with a micro-CTA: "That's it. Start in under 2 minutes."

### 6. Social Proof Deep Dive
- 1-2 customer testimonials with:
  - Real name + title + company
  - Photo (or avatar placeholder)
  - Specific result ("Cut onboarding time by 60%")
- OR: Case study card with metric + link
- OR: Star ratings + review count from G2/Capterra

### 7. Pricing (optional — include if user requests)
- 2-3 tiers max on landing pages
- Highlight recommended tier visually
- Include: tier name, price, billing period, 3-5 bullet features, CTA per tier
- Annual/monthly toggle if applicable
- "No credit card required" or "14-day free trial" trust signal

### 8. FAQ
- 4-6 questions max
- Accordion pattern (expandable)
- Address: pricing concerns, setup difficulty, integrations, data/security, cancellation
- Write answers in 2-3 sentences, conversational tone

### 9. Final CTA Section
- Repeat the primary CTA from hero
- New headline reinforcing urgency or value
- Optional: testimonial snippet alongside the CTA
- High contrast background to break visual rhythm

### 10. Footer
- Product links, legal links (privacy, terms), social icons
- Keep minimal — this is a landing page, not a sitemap

## Copy Guidelines

| Element | Formula | Example |
|---------|---------|---------|
| Headline | [Verb] + [desired outcome] + [without pain] | "Ship features faster without breaking prod" |
| Subheadline | [Product] helps [audience] [benefit] by [mechanism] | "Acme helps dev teams deploy 10x faster with zero-config CI" |
| CTA | [Action verb] + [what they get] | "Start free trial" / "See it in action" |
| Feature title | [Benefit], not [feature] | "Never miss a deadline" not "Task scheduling" |
| Testimonial | [Specific result] + [emotional relief] | "We cut deploy time by 80%. I finally sleep at night." |

## Conversion Patterns
- Every scroll-depth should have a visible CTA (sticky nav CTA or inline)
- Use directional cues (arrows, eye-line of photos) pointing toward CTAs
- Contrast: CTA buttons must pass 4.5:1 against their background
- Reduce friction: "No credit card required", "Setup in 60 seconds", "Cancel anytime"
