# Portfolio / Personal Landing Page Reference

## Required Sections (in order)

### 1. Hero / Introduction
- **Name**: Large, typographically distinctive
- **Title/Role**: One line — "Product Designer", "Full-Stack Engineer", "Creative Director"
- **One-liner**: What you do + for whom. Max 15 words.
- **Primary CTA**: "View my work" / "Get in touch" / "Download resume"
- **Visual**: Professional photo, illustration, or abstract identity element
- **Vibe**: This sets the entire tone — bold, minimal, playful, whatever fits the person

### 2. Selected Work / Case Studies (3-6 items)
- Card or gallery layout
- Each item: thumbnail + project name + one-line role/description
- Hover effect revealing more detail or linking to case study
- Filter by category if 6+ projects (design, dev, writing, etc.)
- Order by impressiveness, not chronology

### 3. About
- 2-3 paragraphs max, first-person voice
- Cover: what you do, your approach/philosophy, what excites you
- Include a photo if not in hero
- Optional: "Currently at [Company]" or "Available for freelance"
- Personality > credentials — show who you are, not just what you've done

### 4. Skills / Services
- If freelancer: list services with 1-line descriptions
- If employee: skill tags or proficiency indicators
- Group by category (Design, Development, Strategy)
- Avoid rating bars (1-5 stars for skills looks amateur)

### 5. Testimonials / Endorsements (optional)
- 1-3 quotes from clients or colleagues
- Include name + relationship ("Former manager at Google")
- Short and specific beats long and vague

### 6. Contact
- Simple form: name, email, message (3 fields max)
- Or: direct email link + social links
- Availability status: "Currently available for projects starting Q2 2026"
- Location / timezone if relevant for remote work

### 7. Footer
- Minimal: social links, copyright, "Built with [tech]" if relevant

## Portfolio-Specific Patterns
- **Case study pages**: If linked, each should have: brief, role, process, outcome, visuals
- **Scroll-driven narrative**: For single-page portfolios, use scroll animations to reveal work progressively
- **Dark mode default**: Portfolios for designers/devs tend to perform better with dark themes (work pops more)
- **PDF resume link**: Always offer a downloadable version
- **GitHub/Dribbble/Behance**: Link to platforms where more work lives

## Copy Guidelines

| Element | Approach | Example |
|---------|----------|---------|
| One-liner | [I verb] + [what] + [for whom] | "I design digital products for startups that move fast" |
| About | First person, conversational, specific | "I've spent 8 years making complex tools feel simple..." |
| Project desc | [What I did] + [the result] | "Redesigned the checkout flow, lifting conversion 23%" |
| CTA | Direct, confident | "Let's work together" / "See the full case study" |
